响应式原理

数据变化时，对应的UI也发生变化。

数据变化后，对应UI不会变的原因

情况1:对象上动态新增的属性。

var vm = new Vue({
  data:{
    a:1
  }
})

// `vm.a` 是响应的

vm.b = 2
// `vm.b` 是非响应的
解决方案

  var vm = new Vue({
    data:{
      a:1,
      b:2,
      c: {}
    }
  })

  // `vm.a`,`vm.b` 是响应的
  vm.$set(vm.c,'d', 2) // vm.c.d 是响应式的
注意：$set的第一个参数不能写 vm
情况2:改的数组内部的值。
            
// 改变数组中，下标为 index 的数据的`name`值
changeName(index, name) {
  this.list[index].name = name
}
解决方案
            
changeName(index, name) {
  this.list = this.list.map((item, i) => {
    if (index === i) {
      return {
        ...item,
        name: name
      }
    } else {
      return item
    }
  })
}

组件间的通信

父组件传递数据给子组件
父组件中通过 `$refs.组件名` 来获得子组件，也就可以调用子组件的属性和方法了。
子组件传递数据给父组件,子组件通过`$emit(eventName)`触发事件，父组件通过`$on(eventName)`监听事件。
子组件调用父组件属性或方法
子组件通过 `$parent` 获得父组件，通过 `$root` 获得最上层的组件。

兄弟,祖先与子组件间通信

方法1:用事件的发布订阅来做(Event Bus)。

var bus = new Vue()

// 组件 A 中触发事件
bus.$emit('id-selected', 1);

// 在组件 B 中监听事件
bus.$on('id-selected', function (id) {
// ...
})

通用的报错处理

推荐 FunDebug: 专业的应用错误监控平台

vm 中报错的处理

import Vue from 'vue'
import {Toast} from 'vant'

Vue.config.errorHandler = function(error, vm, info) {
  var componentName = formatComponentName(vm);
  var propsData = vm.$options && vm.$options.propsData;

  Toast.fail(error)
  console.error(`${componentName}: ${error}`)
  console.error(`更多错误信息: ${info}`)
}

function formatComponentName(vm) {
  if (vm.$root === vm) return 'root';
  var name = vm._isVue ? (vm.$options && vm.$options.name) || (vm.$options && vm.$options._componentTag) : vm.name;
  return (name ? 'component <' + name + '>' : 'anonymous component') + (vm._isVue && vm.$options && vm.$options.__file ? ' at ' + (vm.$options && vm.$options.__file) : '');
}

接口报错的通用处理

import axios from 'axios'
import { Toast } from 'vant'

axios.interceptors.response.use(response => {
  var data = response.data
  var config = response.config
  var errorCode = data.errorCode
  if(errorCode) { 
    Toast(data.errorMessage || '未知错误')
    return Promise.reject()
  }
  return response;
}, error => {
  let errorMessage = "连接到服务器失败"
  if (error && error.response) {
    switch (error.response.status) {
      case 400:
        errorMessage = '错误请求'
        break;
      case 401:
        errorMessage = '未授权，请重新登录'
        break;
      case 403:
        errorMessage = '拒绝访问'
        break;
      // 更多错误信息...
    }
  }

  Toast(errorMessage)
  return Promise.reject(error)
})
